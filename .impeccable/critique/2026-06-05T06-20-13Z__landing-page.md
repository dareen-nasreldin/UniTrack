## Anti-Patterns Verdict

**LLM assessment:** Three structural tells compound each other: (1) three section-opening eyebrow labels (Features, Workflow, Get started) — small-caps, tracked, indigo, above every heading; (2) three identical feat-cards with 40x40px rounded-indigo icon boxes — "identical card grid" pattern; (3) same "No X, no Y. Just Z." rhetorical construction in the hero sub, features sub, and how-it-works sub — three consecutive sections with the same cadence. The 3D CSS mockup with real course data is the strongest counter-tell.

**Deterministic scan:** 5 aphoristic constructions detected (exit 2). Three structural failures: hero sub, features sub, how-it-works sub. Same negation-affirmation cadence repeated.

## Overall Impression

There's a genuinely good page hiding here, strangled by template grammar. The 3D mockup, specific course data, and footer tagline ("Built for students, by a student.") give the page a voice that its own structural scaffold undermines.

## What's Working

1. **The 3D CSS mockup** — shows real course labels (ECE241, MAT291, Amazon SDE Application), inbox alert, and action strip. Not a screenshot or blob — something was built here.
2. **Honest copy** — "Dump it now, sort it later." "See when crunch time hits before it hits you." "Built for students, by a student." These have a point of view.
3. **Semantic accessibility structure** — aria-labelledby on sections, aria-hidden on decorative SVGs, reduced-motion support correct.

## Priority Issues

**[P1] Aphoristic "No X" copy pattern runs three sections in a row.** Hero sub, features sub, how-it-works sub all use negation+affirmation. Confirmed by detector (5 hits). Fix: keep it in one section (features), rewrite the others with positive specific claims. Command: /impeccable clarify.

**[P1] Section eyebrow labels on every section.** `Features`, `Workflow`, `Get started` above every heading. The hero-badge adds a fourth instance. Flatten hierarchy, remove distinctiveness. Fix: drop all .l-section-chip and .l-cta-chip elements; let headings carry weight alone. Command: /impeccable clarify.

**[P1] CTA subtext fails WCAG AA contrast.** `.l-cta-sub` color #475569 on background #0F172A is ~2.5:1, below the 4.5:1 minimum. The CTA section is unreadable for low-vision users. Fix: use #94A3B8 or rgba(255,255,255,0.65). Command: /impeccable audit.

**[P2] Identical feat-card grid with rounded icon boxes.** Three cards, same treatment, same 40px indigo icon squares. Banned pattern. Fix: drop icon boxes, use feature descriptions as labeled callouts into the mockup instead. Command: /impeccable bolder.

**[P2] Logo links to dashboard, not landing.** Unauthenticated users clicking the logo are silently redirected to login. Fix: change href to url_for('landing'). Command: /impeccable harden.

## Persona Red Flags

**Jordan (First-Timer):** Landing page sets expectations (quick capture, course groups, calendar) but doesn't connect them to first-run actions. After registration, no onboarding signal.

**Casey (Distracted Mobile):** Nav buttons (.l-nav-ghost at ~14x32px, .btn-small at min-height 32px) are below 44px touch target minimum. Thumb-accuracy risk.

**Riley (Stress Tester):** "Free for students" appears twice (badge + hero-note). Hero-badge pulsing dot semantically implies "live/online" but the content is a static policy fact.
